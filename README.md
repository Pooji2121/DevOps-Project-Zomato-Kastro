# 🚀 Zomato Clone App Deployment (DevSecOps)

This project demonstrates an end-to-end **DevSecOps pipeline** for deploying a Zomato clone application using modern DevOps tools and cloud technologies.

---

## 🧑‍💻 About the Project

* CI/CD pipeline using Jenkins
* Code quality analysis with SonarQube
* Containerization using Docker
* Deployment using Kubernetes & AWS
* Security scanning with Trivy & OWASP
* Monitoring with Prometheus & Grafana

---

## 🛠️ Tools & Technologies

* GitHub
* Jenkins
* SonarQube
* Docker
* Kubernetes
* AWS (EC2, EKS)
* Trivy
* OWASP Dependency Check
* Prometheus
* Grafana
* ArgoCD

---

## ⚙️ Project Workflow

1. Code pushed to GitHub
2. Jenkins pipeline triggered
3. SonarQube analysis & quality gate
4. Security scanning (Trivy, OWASP)
5. Docker image build
6. Push image to DockerHub
7. Deployment on Kubernetes / AWS
8. Monitoring setup using Prometheus & Grafana

---

## 📂 Project Structure

* `src/` → Application source code
* `Kubernetes/` → Deployment manifests
* `Dockerfile` → Container configuration
* `jenkinsfile` → CI/CD pipeline
